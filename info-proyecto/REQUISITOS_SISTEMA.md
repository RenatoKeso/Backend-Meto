# REQUISITOS DEL SISTEMA

---

## GESTION DE VOLUNTARIOS
El sistema permitirá que los voluntarios, una vez registrados, completen y actualicen su pe fil con información complementaria, incluyendo sus capacidades físicas (por ejemplo: movilidad, resistencia física, capacidad para levantar peso y otras habilidades relevantes para las actividades de voluntariado). 
Cuando una organización publique una actividad, el sistema verificará automáticamente qué voluntarios disponibles cumplen con los requisitos definidos para dicha actividad (por ejemplo, capacidades físicas, edad o certificaciones, si corresponde). Solo estos voluntarios podrán visualizar la opción de postular o aceptar la invitación para participar. La asignación definitiva del voluntario a la actividad será realizada por la organización responsable, una vez que el voluntario confirme su disponibilidad y aceptación.

## ROLES Y AUTENTICACION
El sistema deberá permitir la gestión y asignación de roles a los usuarios del sistema (Central, Jefe de Cuadrilla, Voluntario), de manera que cada usuario acceda únicamente a las funcionalidades correspondientes a su rol. El rol Central será responsable de validar las postulaciones de nuevos voluntarios y activar formalmente sus cuentas, asignándoles el rol de Voluntario. El sistema deberá restringir el acceso a cualquier funcionalidad cuya ejecución no corresponda al rol asignado del usuario.

## GESTION DE ACTIVIDADES
Cuando un usuario registrado o administrador necesite organizar eventos, tareas o actividades programadas de una cuadrilla, el sistema deberá permitir crear, visualizar, modificar y eliminar actividades mediante una interfaz de calendario. Cada actividad debera estar asociada a una cuadrilla especifica e incluir datos como nombre de la actividad, fecha, hora, descripción y estado correspondiente. El sistema almacenará automáticamente la información ingresada y mostrará unicamente las actividades corespondientes a la cuadrilla a la que pertenece el usuario autenticado. Las funciones de creación, modificación y eliminación de actividades estarán disponible únicamente para usuarios autorizados con permisos sobre dicha cuadrilla, mientras que la visualización estará registrada a los integrantes de la misma cuadrilla. Antes de guardar la información, el sistema validará que los datos ingresados estén correctos y completos. Una vez registrada o modificada la actividad esta quedará almacenada en la base de datos y estará disponible para los usuarios autorizados de la cuadrilla correspondiente. 

## GESTION DE BENEFICIADOS
El sistema deberá permitir al personal de la Fundación gestionar la información de las familias beneficiadas. El rol Central tendrá acceso completo al modulo, el jefe de cuadrilla podrá registrar y consultar familias, y el voluntario general de la zona solo podrá consultar información registrada. El usuario debe estar autenticado en el sistema con un rol valido. Para registrar o modificar una familia el usuario debe tener rol Central o jefe de Cuadrilla. Para desactivar una familia el usuario debe tener rol Central únicamente. El sistema valida que el RUT del representante y la dirección no estén previamente registrados y si los datos son validos se registrara la familia marcándola como activa. Si el usuario selecciona consultar el sistema muestra la información completa de la familia incluyendo sus integrantes. Si el usuario selecciona modificar (solo rol Central) el sistema permite actualizar los campos necesarios. Si el usuario selecciona desactivar (solo rol Central) el sistema aplica un borrado lógico marcando la familia como inactiva conservando el registro histórico. La familia queda registrada en el sistema como activa y puede postular a los beneficios y programas de la organización. Toda modificación queda reflejada inmediatamente en el sistema. Un registro de familia no puede eliminarse físicamente de la base de datos, solo puede desactivarse mediante borrado lógico para preservar la trazabilidad histórica.

## GESTION DE DONACIONES 
Gestión de donaciones El sistema deberá permitir a los donantes o al administrador registrar un aporte monetario en el sistema al momento de realizar una transacción económica. El donante podrá elegir ser anónimo ingresando únicamente el monto, o quedar registrado ingresando su RUT, nombre, correo electrónico, el monto. Una vez ingresados los datos, el dónate deberá adjuntar el comprobante de la transferencia bancaria en formato imagen (JPG, PNG) o PDF. El archivo será almacenado en el sistema y la donación quedará en estado pendiente hasta que un administrador la revise internamente. El administrador podrá visualizará el historial de donaciones con su comprobante adjunto y cambiar el estado de cada donación a valida o rechazada según corresponda. Solas las donaciones validas serán consideradas como ingresos efectivos para la organización
